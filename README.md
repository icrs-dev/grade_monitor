# CloudMarking 云阅卷成绩自动监控与多维诊断系统

[![Go Version](https://img.shields.io/github/go-mod/go-version/LuckyMeng-code/grade_monitor)](https://golang.org)
[![Docker Image Size](https://img.shields.io/badge/docker%20image%20size-~30MB-blue)](https://hub.docker.com)
[![GitHub tag](https://img.shields.io/github/v/tag/LuckyMeng-code/grade_monitor?include_prereleases&sort=semver)](https://github.com/LuckyMeng-code/grade_monitor/tags)

**CloudMarking** 是一款为云阅卷平台量身定制的轻量级成绩自动监控、消息通知与多维诊断系统。

本项目后端已由原先的 Python (FastAPI) 完整重构为 **Go (Golang)**。整个后端采用原生静态资源嵌入技术（`go:embed`），将编译好的前端 React 单页面应用直接封包到单个二进制程序中，实现了**零运行时依赖、秒级启动与极低资源消耗**的轻量化服务，非常适合部署在低配 VPS、自建软路由、NAS 或个人电脑上长期常驻挂机。

---

## 🌟 核心特性

- 🚀 **一键服务化部署**：前端 HTML/JS/CSS 静态包直接嵌入 Go 二进制，无需预装 Nginx 或 Python 依赖，开箱即用。
- 📊 **小题多维诊断分析**：提供细化至每道大题/小题的个人得分与全班/全年级得分率的条形图对比，优势弱势科目一目了然。
- 🖼️ **原卷/答题卡流式代理**：安全中转外部扫描的答题卡原图，通过后置代理完美避开现代浏览器对 HTTP/HTTPS 混合内容和跨域（CORS）的拦截阻碍。
- 🔔 **Telegram 成绩实时监控**：基于 Goroutine 承载的常驻异步监控，定时查分并计算哈希。当检测到成绩更新时，向绑定 TG 自动推送排版精致的 HTML 成绩通报。
- 🐳 **多阶段超轻量 Docker 镜像**：基于多阶段构建封装，最终运行镜像基于极简的 Alpine Linux，大小仅有 **30MB 左右**，启动仅需几微秒。

## 🛠️ Linux 一键部署服务化工具 (`install.sh`)

为了极大降低部署门槛，我们提供了一键式安装脚本。在您的 Linux 系统上（支持 Ubuntu, Debian, CentOS 等），只需直接执行以下一行命令即可自动下载、授权并启动部署菜单：

```bash
curl -fsSL https://raw.githubusercontent.com/LuckyMeng-code/grade_monitor/master/install.sh -o install.sh && chmod +x install.sh && sudo ./install.sh
```

对于国内网络环境受限的服务器，可以使用以下国内加速代理命令：

```bash
curl -fsSL https://ghproxy.net/https://raw.githubusercontent.com/LuckyMeng-code/grade_monitor/master/install.sh -o install.sh && chmod +x install.sh && sudo ./install.sh
```

### 部署菜单选项：
1. **方案 1：使用 Docker 一键构建与运行 (强烈推荐)**
   - 自动检测并拉起 Docker，执行多阶段静态包编译，并在后台运行具有重启保护（`--restart unless-stopped`）且端口自定的容器。
2. **方案 2：本地安装 Go、编译并配置为 Systemd 系统服务 (自启动)**
   - 自动识别您的系统架构（如 `x86_64` 或 `aarch64`/树莓派等），极速下载并配置官方 Go 1.21 环境。
   - 编译 Go 二进制，并为您自动注册 `/etc/systemd/system/grade-monitor.service` 系统服务，实现开机自启。
3. **方案 3：仅本地编译**
   - 整理外部依赖并为您生成本地静态 `grade_monitor` 二进制包。

## 💻 开发者本地手动编译指南

如果您希望在本地进行定制开发和编译，请确保您已安装有 [Go](https://golang.org) 和 [Node.js](https://nodejs.org)。

### 1. 编译前端单页应用
```bash
cd frontend

# 安装依赖
npm install

# 前端打包输出到 static/ 目录
npm run build
```

### 2. 编译 Go 后端
回到项目根目录下：
```bash
# 整理拉取 Go 依赖
go mod tidy

# 静态打包编译
go build -o grade_monitor
```
此时，您将得到一个独立的二进制文件 `grade_monitor`（Windows 下为 `grade_monitor.exe`），双击或通过命令行运行 `./grade_monitor` 即可拉起完整系统。

## 📁 配置文件说明 (`config.json`)

系统启动后会自动载入目录下的 `config.json` 进行配置持久化。结构说明如下：

```json
{
  "org_id": "90001",           // 云阅卷平台分配给学校的机构 ID (可在登录页学校列表检索)
  "username": "2028010203",    // 考生的登录学籍号/账号
  "password": "yourpassword",  // 考生的登录密码 (接口获取时会进行掩码脱敏)
  "tg_token": "your_bot_token",// Telegram Bot 令牌 (用于变化推送)
  "tg_chat_id": "your_chat_id",// 接收推送消息的 Telegram 聊天 ID
  "monitor_enabled": false,    // 是否开启自动监控定时轮询
  "monitor_interval": 3600,    // 自动监控间隔时间 (秒，建议不低于 1800)
  "api_key": "autogenerated"   // 首次运行自动生成的管理 API 密钥 (用于敏感接口鉴权)
}
```

## ⚙️ 服务管理常用指令 (Systemd 服务部署下)

当您使用一键脚本安装为 Systemd 服务后，可以使用以下指令对服务进行维护：
* **启动服务**：`systemctl start grade-monitor`
* **停止服务**：`systemctl stop grade-monitor`
* **重启服务**：`systemctl restart grade-monitor`
* **查看运行状态**：`systemctl status grade-monitor`
* **查看实时日志**：`journalctl -u grade-monitor -f`
